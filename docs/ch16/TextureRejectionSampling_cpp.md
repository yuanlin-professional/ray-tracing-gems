# TextureRejectionSampling.cpp 文档

## 1. 变换概述

| 项目 | 内容 |
|------|------|
| **名称** | 纹理拒绝采样 (Texture Rejection Sampling) |
| **用途** | 使用拒绝采样 (rejection sampling) 方法，在纹理上按亮度 (brightness) 成比例地采样坐标 |
| **应用场景** | 发光纹理采样、纹理化光源的光子发射、程序化纹理的重要性采样 |

---

## 2. 数学推导

### 2.1 拒绝采样原理

目标分布 $f(x,y)$ 正比于纹理亮度 $b(x,y) \in [0, 1]$。

拒绝采样步骤：
1. 从提议分布 (proposal distribution)——此处为均匀分布——中采样 $(X, Y)$
2. 生成一个独立的均匀随机数 $U \sim U(0,1)$
3. 若 $U \le b(X, Y)$，接受该样本；否则拒绝并重复

### 2.2 接受概率

每次尝试的接受概率为：

$$
P(\text{accept}) = \int_0^1\int_0^1 b(x, y)\, dx\, dy = \bar{b}
$$

其中 $\bar{b}$ 为纹理的平均亮度。期望的尝试次数为 $1/\bar{b}$。

### 2.3 结果分布

接受的样本 $(X, Y)$ 的分布正比于 $b(X, Y)$，即：

$$
f(X, Y) = \frac{b(X, Y)}{\int\int b(x,y)\, dx\, dy}
$$

## 3. 完整代码与逐行注释

```cpp
do {
    X = u();                              // 在 [0,1) 上均匀采样 X 坐标
    Y = u();                              // 在 [0,1) 上均匀采样 Y 坐标
} while (u() > brightness(texture(X,Y))); // 拒绝采样判断：
                                           // texture(X,Y) 返回纹理颜色
                                           // brightness(...) 计算亮度，范围 [0,1]
                                           // u() 生成新的均匀随机数用于接受/拒绝判断
                                           // 若 u() > brightness，则拒绝，继续循环
                                           // 若 u() <= brightness，则接受，退出循环
// Brightness is [0,1].
// 亮度范围假定为 [0,1]
```

---

## 4. 输入与输出

### 输入参数

| 参数 | 类型 | 范围 | 说明 |
|------|------|------|------|
| `u()` | 函数 | 返回 $[0,1)$ | 均匀随机数生成器，每次调用返回独立随机数 |
| `texture(X,Y)` | 函数 | — | 纹理查找函数，返回 $(X,Y)$ 处的颜色值 |
| `brightness(c)` | 函数 | $[0, 1]$ | 将颜色转换为亮度值（需归一化到 $[0,1]$） |

### 输出

| 输出 | 类型 | 说明 |
|------|------|------|
| `X` | `float` | 采样的纹理 U 坐标 $\in [0,1)$ |
| `Y` | `float` | 采样的纹理 V 坐标 $\in [0,1)$ |

---

## 5. 应用场景

- **纹理化面光源**：对带有发光纹理的面光源进行亮度加权采样，使亮的区域被更频繁地采样
- **星空贴图**：在星空环境贴图中按星星的亮度采样方向
- **投影纹理**：对投影灯 (gobo / projector light) 的纹理进行重要性采样
- **程序化纹理**：对分析表达式定义的纹理进行采样（无需预计算 MipMap 或 CDF）

## 6. 与相关变换的对比

| 方法 | 预处理 | 采样效率 | 采样精度 | 适用场景 |
|------|--------|---------|---------|---------|
| **拒绝采样 (本方法)** | 无 | 取决于平均亮度 $\bar{b}$；$\bar{b}$ 越低效率越差 | 连续（不限纹素） | 小纹理或动态纹理 |
| **MipMap 层级采样** (`SampleMipMap.cpp`) | MipMap 构建 | $O(\log N)$，始终高效 | 纹素级别 | 静态纹理 |
| **CDF 表** | $O(N^2)$ 构建 | $O(\log N)$ 二分查找 | 纹素级别 | 静态大纹理 |
| **分段常数** (`samplePiecewiseConstantArray.cpp`) | $O(N)$ CDF 构建 | $O(\log N)$ | 段级别 | 一维分布 |

> **效率警告**：当纹理的平均亮度很低时（如大部分区域很暗，仅少数亮点），拒绝率非常高，效率极差。例如，若平均亮度为 1%，则平均需要 100 次尝试才能接受一个样本。此时应使用 `SampleMipMap` 或 CDF 方法。拒绝采样的优势在于零预处理开销和连续精度的采样位置。
